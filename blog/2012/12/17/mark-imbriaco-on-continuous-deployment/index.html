Link: Mark Imbriaco on continuous deployment

December 17, 2012

Code that has been written but not yet deployed is very similar to inventory. You’ve paid the cost to develop the software, but are not yet getting any of the benefit from it. Shipping that code to production sooner means that you and your customers can benefit from it much faster. This is a pretty serious competitive advantage for companies that deliver features faster than their competitors.

-- Mark Imbriaco
